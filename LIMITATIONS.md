# Limitations

This repo uses sample receipts only.

It does not integrate with a live model, agent runtime, production workflow, or external enforcement system.

It does not intercept real unsafe actions.

It does not prove prevention, adoption, compliance, certification, or standardisation.

It demonstrates one bounded mechanism:

If a refusal occurs, it can be recorded, policy-bound, hash-chained, and replayed for later inspection.

## Claim boundary

refusal-receipt-chain is an inspectable proof surface for post-hoc verification of refusals. It does not claim adoption, compliance, standardisation, prevention, or originality.
